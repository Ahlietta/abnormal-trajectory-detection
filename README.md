# ATC 行人轨迹异常检测系统
## 📌 项目简介

本项目基于 ATC（*Asia and Pacific Trade Center*）商场行人轨迹数据集，构建了一套 **「无监督全局异常度 + 多维度专项评分」** 结合的异常行为检测系统。通过提取 23 维运动学、形态学与空间特征，实现对 **奔跑、徘徊、闯入禁区** 三类典型异常行为的精准识别，同时提供桌面可视化交互、RESTful API 接口、批量离线处理、实验分析全链路能力。

评分规则采用**数据驱动的权重优化方案**：通过单特征 AUC 量化区分度，结合特征相关性去冗余，最终权重完全由真实数据分布决定，相比人工设定更贴合 ATC 商场实际场景。

## ✨ 核心特性

- **科学的评分体系**：单特征 AUC 区分度量化 + 相关性去冗余 + AUC 归一化加权，替代纯人工经验权重
- **双级异常判定**：DBSCAN 无监督聚类输出整体异常度 + 三类异常独立专项评分，兼顾全局识别与细分类
- **多形态交互入口**
  - 桌面 GUI：鼠标手绘轨迹，实时 / 批量双模式检测，可视化展示结果
  - Web 接口：FastAPI 实现标准 HTTP 接口，可对接前端、监控大屏
  - Python SDK：直接调用核心检测类，灵活集成到业务流程
- **完整实验链路**：从数据预处理、特征工程、模型训练到特征贡献分析的全流程 Notebook，可复现、可迭代
- **工程化实现**：并发安全的接口设计、完善的异常处理、跨平台中文兼容

## 🧠 算法原理

### 1. 特征体系

构建 23 维多维度特征，分为四大类：

- **运动学特征**：平均速度、最大速度、速度标准差、速度变异系数、最大加速度、加速度标准差
- **方向特征**：平均方向变化、最大方向变化、方向变化标准差、转向频率、方向熵、方向反转率
- **轨迹形态特征**：轨迹总长度、起止点距离、迂回度、低速占比、轨迹时长
- **空间特征**：轨迹中心坐标、到禁区最小距离、禁区内停留占比、朝向偏差

### 2. 两级检测架构

采用「**全局无监督聚类 + 专项加权评分**」的两级检测框架：

1. **全局异常度**：基于纯正常样本训练 DBSCAN 聚类模型，计算待测轨迹到最近核心样本的距离，衡量整体异常程度
2. **专项异常评分**：针对奔跑、徘徊、闯入禁区三类异常，分别筛选高区分度特征，通过数据驱动的权重加权融合，输出 0~1 的异常分数
3. **阈值报警**：单项分数超过对应阈值时，触发对应类别的异常报警

### 3. 权重优化方法

- **区分度量化**：以单特征 AUC 作为指标，量化每个特征区分「正常 vs 某类异常」的能力
- **去冗余处理**：计算特征间相关性，对相关系数 > 0.8 的高冗余特征施加权重惩罚，避免单一维度过度主导评分
- **归一化加权**：有效特征的 AUC 值归一化后作为最终权重，完全由数据分布决定，避免人工经验偏差

## ⚙️ 自定义配置

### 调整报警阈值

检测时传入自定义阈值字典，灵活调整灵敏度：

```python
custom_thresholds = {
    'running': 0.5,    # 奔跑异常阈值
    'loitering': 0.55, # 徘徊异常阈值
    'trespassing': 0.4 # 闯入禁区阈值
}
result = detector.detect(x, y, custom_thresholds=custom_thresholds)
```

### 修改禁区配置

修改 `data/processed/forbidden_zone.npy` 中的禁区中心坐标，或在检测器初始化后调整对应属性。

## ❓ 常见问题

1. **启动提示找不到模型文件**

   检查运行目录是否正确，需在 `src` 目录下执行程序，确保相对路径 `../data/processed/` 可正确定位。

2. **GUI 画布中文显示为方块**

   已内置多平台中文字体适配，若仍异常可安装对应系统的中文字体（Windows 安装微软雅黑，Linux 安装文泉驿微米黑）。

3. **Web 服务启动端口报错**

   更换端口号，或关闭占用对应端口的程序；本地调试可将 `0.0.0.0` 改为 `127.0.0.1` 降低权限要求。

4. **轨迹分数异常偏低 / 偏高**

   确认坐标单位为毫米，与 ATC 数据集单位一致；可通过调整报警阈值适配不同安防等级。
